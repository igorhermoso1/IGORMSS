Urruneko saioak: administrar desde lejos

No siempre estás delante del servidor. Las sesiones remotas te dejan controlarlo desde otro equipo: por texto con SSH, por web con Webmin, o con escritorio gráfico mediante VNC y AnyDesk.

Analogía

Tres formas de operar a distancia. SSH es una línea de teléfono segura a la sala de máquinas: solo voz (texto), pero rápida y cifrada. Webmin es un panel de mandos por la web: botones en vez de comandos. VNC/AnyDesk es una cámara que ve la pantalla y unas manos que mueven el ratón: ves el escritorio tal cual.

HerramientaTipoPuertoCuándo usarla

2. SSH (OpenSSH) — el núcleo

SSH (Secure Shell) abre una sesión de terminal en una máquina remota de forma cifrada. Trabaja sobre TCP en el puerto 22, con modelo cliente/servidor. La implementación libre estándar es OpenSSH.

2.1 · Instalar el servidor (5_AS)

En u18zerbitzaria (192.168.4.101), que será el servidor:

u18zerbitzaria
$ sudo apt update && sudo apt install -y openssh-server
$ sudo systemctl status ssh     # debe aparecer "active (running)"
$ sudo systemctl enable ssh     # que arranque al encender

2.2 · Conectar desde el cliente

Desde u20zerbitzuak, abre sesión como el usuario mss2 del servidor:

u20zerbitzuak — cliente
$ ssh mss2@192.168.4.101
# la primera vez acepta la huella del servidor (fingerprint) escribiendo "yes"

2.3 · Copiar ficheros: scp

SSH también transporta ficheros con scp (copia segura):

u20zerbitzuak — cliente
# subir un fichero local al servidor
$ scp informe.txt mss2@192.168.4.101:/home/mss2/
# descargar del servidor al cliente
$ scp mss2@192.168.4.101:/home/mss2/datos.csv .

2.4 · Claves en vez de contraseña (recomendado)

u20zerbitzuak — cliente
$ ssh-keygen -t ed25519             # genera tu par de claves
$ ssh-copy-id mss2@192.168.4.101     # instala tu clave pública en el servidor
$ ssh mss2@192.168.4.101            # ahora entra sin pedir contraseña

Simulador · Constructor de comando ssh

Rellena los campos y verás el comando montarse en vivo, con cada parte explicada. Así entiendes qué significa cada trozo antes de escribirlo en la terminal.

Errores comunes

1. "Connection refused". El servicio ssh no está arrancado o el cortafuegos cierra el 22.
2. Olvidar el usuario. ssh 192.168.4.101 intenta entrar con tu usuario local; casi siempre quieres usuario@host.
3. Cambiar el puerto en el servidor y no en el cliente. Si en sshd_config pones Port 2222, conecta con ssh -p 2222 ….

3. Webmin — administrar por web (6_AS)

Webmin es un panel de administración que se usa desde el navegador en https://IP:10000. Permite configurar servicios (incluido el propio SSH) sin tocar la terminal.

u18zerbitzaria
# método actual con repositorio oficial (webmin.com)
$ curl -o setup-repos.sh https://raw.githubusercontent.com/webmin/webmin/master/setup-repos.sh
$ sudo sh setup-repos.sh
$ sudo apt install -y webmin

Después, desde un navegador (en el propio servidor o desde un cliente de la LAN), entra en:

navegador
https://192.168.4.101:10000
# inicia sesión con tu usuario del sistema (mss2). Acepta el aviso de certificado.

Para tocar SSH desde Webmin: Servidores → Servidor SSH → Ficheros de configuración. Ahí editas sshd_config con formularios en vez de a mano. También puedes cambiar idioma y tema en Webmin → Change Language and Theme.

4. Escritorio gráfico: VNC y AnyDesk (6_AS / 7_AS)

4.1 · VNC en la LAN

VNC comparte el escritorio gráfico. El servidor arranca una sesión y el cliente la ve con un visor. Cada sesión usa un "display" (:1 → puerto 5901).

u18zerbitzaria
# los apuntes usan vnc4server; en sistemas actuales suele instalarse tigervnc
$ sudo apt install -y tigervnc-standalone-server
$ vncserver                  # crea la sesión :1 y pide una contraseña
$ vncserver -list             # ver sesiones activas

Desde el cliente, con un visor VNC, conecta a 192.168.4.101:1 (o :5901).

4.2 · AnyDesk (acceso fácil, incluso por Internet)

AnyDesk es gráfico y atraviesa NAT a través de su servicio de relay: no hace falta abrir puertos. Se descarga de anydesk.com.

u18zerbitzaria
$ sudo apt install -y ./anydesk_*.deb   # instalar el paquete descargado
$ anydesk                       # muestra tu dirección AnyDesk (un número)

En el otro equipo, abre AnyDesk e introduce esa dirección para conectar.

Nota · ¿cuál elijo?

Para administrar un servidor: SSH (ligero y seguro). Para configurar con clics sin instalar nada en el cliente: Webmin. Para ver el escritorio en la LAN: VNC. Para ayudar a alguien a distancia por Internet sin tocar el router: AnyDesk.

5. Ejercicios

Nivel 1 · Reproducir Tu primer ssh ›

Instala openssh-server en u18zerbitzaria y conéctate desde u20zerbitzuak con ssh mss2@192.168.4.101.

Cómo sabes que está bien: tras aceptar la huella e introducir la contraseña, el prompt cambia al nombre del servidor.

Nivel 2 · Modificar Cambiar el puerto y copiar un fichero ›

Cómo sabes que está bien: el simulador muestra ssh -p 2222 mss2@192.168.4.101 y el fichero aparece en el home del servidor.

Nivel 3 · Crear Webmin operativo (6_AS) ›

Cómo sabes que está bien: ves el panel de Webmin y puedes abrir el fichero de configuración de SSH desde la web.

Nivel 4 · Desafío Sin contraseña + sesión gráfica ›

Configura acceso SSH por clave (ssh-keygen + ssh-copy-id) y monta una sesión gráfica con VNC o AnyDesk hacia el servidor. Sin solución proporcionada.

6. Autoevaluación

Comprueba el Tema 4

—

¿Qué puerto y transporte usa SSH por defecto?

SSH va sobre TCP en el 22. El 10000 es Webmin; el 68 es DHCP cliente.

Quieres copiar un fichero al servidor de forma cifrada. ¿Qué usas?

scp copia sobre el canal seguro de SSH. cp es local y ping solo prueba conectividad.

¿En qué URL accedes a Webmin?

Webmin escucha por HTTPS en el puerto 10000.

Necesitas ayudar gráficamente a un equipo en otra red, sin abrir puertos en su router. ¿Qué encaja mejor?

AnyDesk usa un relay que atraviesa NAT; VNC directo requeriría exponer el puerto.
