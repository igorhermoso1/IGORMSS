DHCP: direcciones automáticas

Configurar la IP a mano en cada equipo no escala. El DHCP reparte direcciones (y máscara, puerta de enlace y DNS) automáticamente cuando un cliente se conecta. Aquí montas el servidor isc-dhcp-server sobre Ubuntu.

1. Qué hace el DHCP

Analogía

La recepción de un hotel. Llegas sin habitación asignada; recepción mira qué habitaciones quedan libres, te da una llave (tu IP) durante tu estancia (el lease) y apunta que es tuya. Si vuelves, intenta darte la misma. Cuando te vas, la habitación queda libre para otro. El DHCP es esa recepción para la red.

El servidor DHCP entrega de forma automática la configuración de red de un cliente. Trabaja sobre UDP, puertos 67 (servidor) y 68 (cliente). Lo que reparte:

ObligatorioOpcional (options)
Dirección IP + máscaraPuerta de enlace (routers)
Tiempo de cesión (lease time)Servidores DNS
Renovación / reconexiónNombre de dominio

2. El intercambio DORA

Cuando un cliente arranca sin IP, cliente y servidor hablan en cuatro mensajes. Se conoce por sus iniciales: DORA.

  • Discover

    El cliente grita por broadcast "¿hay algún DHCP?". Aún no tiene IP, así que pregunta a toda la red.

  • Offer

    El servidor responde con una propuesta: "te ofrezco esta IP, con esta máscara, gateway y DNS".

  • Request

    El cliente acepta y pide formalmente esa IP (otra vez por broadcast, por si hubiera varios servidores).

  • Acknowledge

    El servidor confirma, fija el lease y registra la pareja MAC ↔ IP.

topologia — DORA
  cliente (u20zerbitzuak)            servidor (u18zerbitzaria)
       │ ── DISCOVER (broadcast) ───────────▶ │  busca IP libre en el range
       │ ◀──────────── OFFER (192.168.4.171) ─│
       │ ── REQUEST (quiero la .171) ───────▶ │
       │ ◀──────────── ACK + lease ───────────│  apunta MAC ↔ IP
       ▼
   ya tiene IP, máscara, gateway y DNS

Simulador · Asignador de leases

Define el rango y conecta clientes. El servidor entrega la primera IP libre del range, recuerda cada MAC (si vuelve, le da la misma) y muestra el lease. Si agotas el rango verás el error real de un DHCP sin direcciones.

MACIP asignadaTipo
Errores comunes

1. Range que pisa IPs fijas. Si el servidor (.101) o el gateway (.1) caen dentro del range, habrá conflictos. El range debe ser un tramo libre.
2. Dos DHCP en la misma LAN. Por eso en el aula desconectas el cable de red: así solo responde tu servidor y no el del centro.
3. No declarar la interfaz. Si isc-dhcp-server no sabe en qué tarjeta escuchar, no sirve ninguna IP.

3. Instalación paso a paso (Ubuntu · isc-dhcp-server)

Servidor en u18zerbitzaria (192.168.4.101). Cliente de prueba: u20zerbitzuak. Usa el range que te indique el profesor (cada alumno tiene el suyo para no chocar: 4_AS_IP_Taldeak).

3.1 · Instalar el servidor

u18zerbitzaria
$ sudo apt update && sudo apt install -y isc-dhcp-server

3.2 · Decir en qué interfaz escucha

Edita /etc/default/isc-dhcp-server y pon el nombre de tu tarjeta (míralo con ip a, p. ej. enp0s3):

/etc/default/isc-dhcp-server
INTERFACESv4="enp0s3"

3.3 · Configurar el ámbito (scope)

Edita /etc/dhcp/dhcpd.conf. Comenta el bloque de opciones comunes que viene de ejemplo y añade tu subred con el range y las opciones:

u18zerbitzaria
$ sudo nano /etc/dhcp/dhcpd.conf
/etc/dhcp/dhcpd.conf
subnet 192.168.4.0 netmask 255.255.255.0 {
    range 192.168.4.171 192.168.4.175;          # tu tramo (IKUS ARBELA)
    option subnet-mask 255.255.255.0;
    option routers 192.168.4.1;                 # puerta de enlace
    option broadcast-address 192.168.4.255;
    option domain-name-servers 192.168.4.101, 8.8.8.8;  # tu DNS + Google
    option domain-name "josebabilbao.net";
    default-lease-time 600;
    max-lease-time 7200;
}

# Reserva por MAC (siempre la misma IP a un equipo concreto)
host w10zerbitzuak {
    hardware ethernet 08:00:27:ab:cd:ef;
    fixed-address 192.168.4.180;
}

3.4 · Arrancar y comprobar

Antes de iniciar el servicio, desconecta el cable de red del aula para que solo responda tu servidor.

u18zerbitzaria
$ sudo dhcpd -t -cf /etc/dhcp/dhcpd.conf   # comprobar sintaxis
$ sudo systemctl restart isc-dhcp-server
$ sudo systemctl status isc-dhcp-server
$ journalctl -u isc-dhcp-server -e      # ver los mensajes DORA
Nota · servicio o service

En Ubuntu 18/20 también vale sudo service isc-dhcp-server restart. Las cesiones concedidas se ven en /var/lib/dhcp/dhcpd.leases.

3.5 · Probar desde el cliente

En u20zerbitzuak, pon IPv4 en Automático (DHCP), desconecta y reconecta la red. Después comprueba qué IP recibió y que está dentro del range:

u20zerbitzuak — cliente
$ ip a                          # ¿IP dentro de 192.168.4.171–175?
$ sudo dhclient -r && sudo dhclient -v   # forzar liberar y pedir lease

Tras la prueba, vuelve a poner la IP fija del cliente y reconecta el cable del aula.

4. Ejercicios

Nivel 1 · Reproducir Repartir leases en el simulador ›

Con el range por defecto (.171–.175), conecta clientes hasta llenarlo. Observa el mensaje cuando se agota.

Cómo sabes que está bien: entrega 5 IPs (.171 a .175) y el 6º cliente recibe "sin direcciones libres".

Nivel 2 · Modificar Reserva por MAC ›

En el simulador, conecta el cliente con MAC 08:00:27:ab:cd:ef (la reservada). Comprueba que recibe siempre 192.168.4.180 aunque no esté en el range.

Cómo sabes que está bien: esa MAC obtiene la .180 y aparece marcada como "reserva".

Nivel 3 · Crear Tu servidor DHCP real (4_AS) ›

Monta isc-dhcp-server en u18zerbitzaria con tu range, gateway, máscara y DNS (tu servidor + 8.8.8.8). Cliente en automático recibe IP del range.

Cómo sabes que está bien: en el cliente, ip a muestra una IP de tu range y resuelve nombres con tu DNS.

Nivel 4 · Desafío DHCP + DNS juntos ›

Haz que el DHCP entregue como DNS tu servidor del Tema 2, y que un cliente que recibe IP por DHCP pueda hacer ping w10zer.tudominio.net. Sin solución proporcionada.

5. Autoevaluación

Comprueba el Tema 3

—

¿Cuál es el orden correcto del intercambio DHCP?

DORA: el cliente descubre, el servidor ofrece, el cliente solicita y el servidor confirma.

¿Qué protocolo y puertos usa el DHCP?

DHCP va sobre UDP: 67 en el servidor, 68 en el cliente. El 53 es DNS y el 80 es HTTP.

¿Por qué en el aula se desconecta el cable antes de arrancar tu DHCP?

Varios DHCP en la misma LAN se pisan; aislando tu segmento garantizas que tus clientes piden a tu servidor.

¿Qué consigues con una declaración host (hardware ethernet + fixed-address)?

Es una reserva: ata una MAC a una IP fija, útil para impresoras o servidores.
