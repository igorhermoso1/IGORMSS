Máquinas virtuales y direcciones IP

Antes de instalar ningún servicio necesitas un laboratorio. Aquí montas las 7 máquinas del aula sobre VirtualBox y les das una dirección fija dentro de la red 192.168.4.0/24.

1. Qué es virtualizar

Analogía

Un edificio de pisos. El edificio es tu ordenador real (la estructura, la luz, el agua: CPU, RAM, disco). Cada piso es una máquina virtual: tiene sus paredes, su puerta y su inquilino (un sistema operativo) que vive como si tuviera una casa entera, sin enterarse de que comparte cimientos con los vecinos. El portero que reparte luz y agua a cada piso es el hipervisor: VirtualBox.

Una máquina virtual (VM) es un ordenador completo emulado por software dentro de tu ordenador real. Distinguimos dos papeles:

TérminoQué esEn el aula
Anfitrión (host)La máquina física donde se ejecuta todoEl PC del aula
Invitado (guest)Cada sistema operativo virtualizadoUbuntu, Windows…
HipervisorEl programa que crea y reparte recursos a las VMsVirtualBox

Diagrama: una red dentro de tu PC

topologia — laboratorio
            PC del aula (ANFITRIÓN)  ──  hipervisor: VirtualBox
   ┌──────────────────────────────────────────────────────┐
   │  [u18zerbitzaria]  [u20zerbitzaria]   ← SERVIDORES     │
   │  [w10zerbitzuak]   [u20zerbitzuak]    ← serv. servicios│
   │  [w11] [u22] [u24]                    ← CLIENTES        │
   └───────────────┬──────────────────────────────────────┘
                   │  Adaptador puente
            ┌──────┴───────┐
            │  LAN del aula │  192.168.4.0/24   gateway .4.1
            └──────────────┘  (profesor: 192.168.4.10)
Errores comunes

1. NAT en vez de puente. Con el adaptador en modo NAT las VMs no se ven entre sí ni el profesor las alcanza. Para un laboratorio de servicios necesitas adaptador puente.
2. Dos VMs con la misma IP. Si clonas una máquina y no cambias su IP, ambas chocan y ninguna conecta bien.
3. Olvidar las Guest Additions. Sin ellas no tendrás pantalla completa ni carpetas compartidas.

2. Instalación paso a paso

2.1 · Preparar las ISO

Crea una carpeta para guardar todas las imágenes de los sistemas operativos y cópialas desde el equipo del profesor por FTP.

$ mkdir -p ~/ISO
# El profesor publica las ISO por FTP en 192.168.4.10 (usuario mss2)

2.3 · Adaptador de red en puente

Para cada VM: Configuración → Red → Adaptador 1 → Conectado a: Adaptador puente. Así cada máquina obtiene una dirección dentro de la LAN del aula y todas se ven entre sí.

2.4 · Poner la IP fija (Ubuntu Server, netplan)

Ubuntu Server usa netplan. Edita el fichero de configuración y aplica. Sustituye enp0s3 por el nombre real de tu interfaz (míralo con ip a) y el último octeto por el que te asigne el profesor.

u20zerbitzaria — invitado
$ ip a                       # ver el nombre de la interfaz
$ sudo nano /etc/netplan/01-netcfg.yaml
01-netcfg.yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.4.101/24]
      gateway4: 192.168.4.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
u20zerbitzaria — invitado
$ sudo netplan apply
$ ip a                       # comprobar que la IP es la esperada
$ ping -c2 8.8.8.8            # comprobar salida a Internet
Nota · ¿por qué así?

En netplan reciente gateway4 está obsoleto. La forma moderna es routes: [{ to: default, via: 192.168.4.1 }]. En Ubuntu 18/20 ambas funcionan; usamos gateway4 por ser lo que verás en muchos apuntes. La indentación en YAML es con espacios, nunca tabuladores.

2.5 · Poner la IP fija (Windows)

En las VMs Windows: Configuración de red → Cambiar opciones del adaptador → clic derecho en la conexión → Propiedades → IPv4 → Usar la siguiente dirección IP.

CampoValor
Dirección IP192.168.4.X (te lo asigna el profesor)
Máscara de subred255.255.255.0
Puerta de enlace192.168.4.1
DNS preferido / alternativo8.8.8.8 / 8.8.4.4
w10zerbitzuak — comprobación (cmd)
> ipconfig /all
> ping 192.168.4.1

4. Autoevaluación

—

¿Qué papel cumple VirtualBox en el laboratorio?

El hipervisor es el software que virtualiza: reparte CPU, RAM y disco del anfitrión entre los invitados.

Quieres que las VMs se vean entre sí y que el profesor las alcance en la LAN del aula. ¿Qué modo de red eliges?

El puente conecta la VM directamente a la red física, así obtiene una IP de la LAN y es visible para el resto.

En la red del aula, ¿cuál es la máscara y la puerta de enlace?

Máscara /24 = 255.255.255.0. La puerta de enlace es .4.1; el .4.10 es el equipo del profesor, no el router.

Tras editar el netplan, ¿qué comando aplica los cambios?

netplan lee el YAML y reconfigura la red con netplan apply.
