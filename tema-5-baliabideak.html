Compartir recursos y proxy-caché

Dos servicios distintos pero del mismo tema. Samba comparte carpetas entre Linux y Windows. Squid es un proxy-caché que controla y acelera la salida a Internet de los clientes.

1. Samba — compartir carpetas (8_AS)

Analogía

Un archivador común en la oficina. En vez de que cada uno guarde copias en su mesa, hay un mueble compartido al que todos acceden con su llave. Samba convierte una carpeta de tu servidor Linux en ese archivador, y hace que máquinas Windows y Linux lo vean por igual hablando el protocolo SMB.

1.1 · Instalar Samba

u18zerbitzaria
$ sudo apt update && sudo apt install -y samba

1.2 · Declarar el recurso compartido

Edita /etc/samba/smb.conf y añade al final un bloque con el nombre del recurso, la ruta y los permisos:

/etc/samba/smb.conf
[elkarbanatu]
    path = /home/mss2/Mahaigaina
    valid users = mss2
    read only = no
    browseable = yes

1.3 · Crear el usuario Samba y reiniciar

El usuario debe existir en el sistema. Luego se le da una contraseña de Samba (independiente de la del sistema):

u18zerbitzaria
$ sudo smbpasswd -a mss2          # pide y fija la contraseña Samba
$ sudo systemctl restart smbd     # (también vale: sudo service smbd restart)

1.4 · Acceder al recurso

Desde otra máquina de la LAN:

SistemaCómo conectar
Linux (Archivos)Otras ubicaciones → smb://192.168.4.101
Windows (Explorador)\\192.168.4.101\elkarbanatu
Errores comunes

1. Usuario sin contraseña Samba. Existir en el sistema no basta; hay que ejecutar smbpasswd -a.
2. Permisos del directorio. Si mss2 no tiene permiso de escritura en la carpeta real, Samba tampoco lo dará.
3. Cortafuegos. En Windows, el firewall puede bloquear el acceso al recurso.

2. Squid — proxy-caché (9_AS)

Analogía

La recepción de un edificio que recoge tus paquetes. En vez de que cada vecino baje a la calle, recepción sale por todos, guarda copia de lo más pedido (la caché, que acelera) y puede negarse a aceptar ciertos envíos (las reglas/ACL). Squid es esa recepción para el tráfico web de los clientes.

Un proxy se interpone entre los clientes e Internet: centraliza la salida, cachea contenido y aplica reglas de acceso. Squid escucha por defecto en el puerto 3128.

2.1 · Instalar y configurar

u20zerbitzuak — servidor proxy
$ sudo apt update && sudo apt install -y squid
$ sudo cp /etc/squid/squid.conf /etc/squid/squid.conf.bak   # copia de seguridad
$ sudo nano /etc/squid/squid.conf

Define quién puede usar el proxy y qué se bloquea. Las reglas http_access se evalúan en orden, y gana la primera que coincide:

/etc/squid/squid.conf
http_port 3128
visible_hostname proxy.josebabilbao.net

# quién es la red local
acl localnet src 192.168.4.0/24
# dominios que queremos bloquear
acl bloqueados dstdomain .facebook.com .youtube.com

http_access deny bloqueados
http_access allow localnet
http_access deny all
u20zerbitzuak — servidor proxy
$ sudo systemctl restart squid
$ sudo systemctl status squid

2.2 · Apuntar el cliente al proxy

En el navegador del cliente (o en la configuración de red), pon como proxy HTTP la IP del servidor Squid y el puerto 3128. A partir de ahí, todo su tráfico web pasa por Squid y obedece las reglas.

Simulador · Evaluador de reglas de Squid

Edita las reglas si quieres, indica desde qué IP y a qué dominio quiere ir un cliente, y el simulador aplica las http_access en orden hasta la primera que coincide. Cambia el orden de las líneas y verás por qué importa.

3. Ejercicios

Nivel 1 · Reproducir Probar las reglas ›

Con las reglas por defecto del simulador, evalúa: 192.168.4.50 → google.com, 192.168.4.50 → youtube.com y 10.0.0.5 → google.com.

Cómo sabes que está bien: google permitido, youtube bloqueado y el cliente de fuera de la LAN denegado por deny all.

Nivel 2 · Modificar El orden importa ›

En el simulador, mueve http_access allow localnet por encima de http_access deny bloqueados y evalúa de nuevo youtube desde la LAN.

Cómo sabes que está bien: ahora youtube se permite, porque gana la primera regla que coincide (allow localnet). Demuestra por qué el bloqueo debe ir antes.

Nivel 3 · Crear Samba + Squid reales ›

Monta el recurso Samba elkarbanatu accesible desde un cliente, y un Squid que deje navegar a la LAN pero bloquee un dominio a tu elección.

Cómo sabes que está bien: abres el recurso compartido y escribes en él; y el cliente, con el proxy puesto, navega pero no entra en el dominio bloqueado.

Nivel 4 · Desafío Bloqueo por horario ›

Investiga las ACL de tipo time en Squid y crea una regla que bloquee un dominio solo en horario lectivo. Sin solución proporcionada.

¿Qué comando da contraseña de Samba a un usuario existente?

smbpasswd -a añade y fija la contraseña Samba; passwd cambia solo la del sistema.

¿En qué puerto escucha Squid por defecto?

Squid usa el 3128 por defecto. El 445 es SMB/Samba y el 22 es SSH.

Las reglas http_access de Squid se aplican...

Squid evalúa de arriba abajo y se detiene en la primera coincidencia. Por eso el orden es crítico.

Con deny bloqueados antes de allow localnet, un cliente de la LAN pide youtube.com (en bloqueados). ¿Qué pasa?

Gana la primera coincidencia: deny bloqueados se evalúa antes y corta el acceso.
